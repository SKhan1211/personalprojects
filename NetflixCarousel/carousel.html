Things I've Learned with this Mini-Project

  • I can use max-width 100% on the smallest elements of a container to make the parent fit 100% of the viewport responsively
  • The flex CSS property is shorthand for flex-grow, flex-shrink, and flex-basis all in one
  • The general rule to achieve the siblings moving outward is as follow:
    • When hovering over parent container, shift all of the items inside the container to the left
    • Use the general sibling combinator to make the items positioned after the hovered items move to the right
    • Get super specific so a hovered item isn't translated like the rest of the items
  • The general sibling combinator (~) seperates two selectors and matches the second element only if it follows the first element as long as both are children of the same element
  • MDN docs can provide more information about the general sibling combinator with examples
  • Ordering matters, first on hover or active we translate the whole container to the left
  • Next, on hover or active individual items we translate items after the hovered/selected one using the general sibling combinator to the right
  • Lastly, we scale the hovered/selected image to make it bigger and take more space
  • If the order doesn't follow as specified in the stylesheet, it won't work correctly since previous styles will get overwritten not in the order we want to achieve
